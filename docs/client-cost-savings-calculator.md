# Client Cost Savings Calculation Document

Use this document to estimate and explain the cost savings generated by a
self-hosted LiteLLM, Ollama, and adaptive-routing pilot. It is designed for a
client handoff: keep the assumptions visible, separate measured values from
estimates, and attach the raw pilot metrics when possible.

## Executive Summary

| Item | Baseline | Pilot | Difference |
| --- | ---: | ---: | ---: |
| Monthly LLM requests | TBD | TBD | TBD |
| Monthly input tokens | TBD | TBD | TBD |
| Monthly output tokens | TBD | TBD | TBD |
| Paid cloud LLM cost | TBD | TBD | TBD |
| Local/cloud hosting cost | 0 | TBD | -TBD |
| Net monthly cost | TBD | TBD | TBD |
| Net monthly savings | 0 | TBD | TBD |
| Provider 429 errors | TBD | TBD | TBD |
| Avoided 429 errors | 0 | TBD | TBD |

Decision rule:

```text
net_monthly_savings =
  baseline_monthly_llm_cost
  - pilot_monthly_llm_cost
  - incremental_hosting_cost
  - incremental_operations_cost
```

A positive value means the pilot is cheaper than the baseline after including
the extra hosting and operations cost.

For a client engagement, copy the example JSON to a dated client file, update

```powershell
Copy-Item `
  .\docs\smb-llm-pilot\cost-savings-input.example.json `
  .\docs\smb-llm-pilot\client-cost-savings-YYYY-MM-DD.json
```

The JSON fields that normally change are:

- `pricing_date`;
- baseline and route `input_price_per_1m`;
- baseline and route `output_price_per_1m`;
- route `share` values from router metrics;
- fixed monthly costs for OVHcloud, Hetzner, AWS, Azure, or local hardware;
- observed `baseline_429_count` and `pilot_429_count`.

Do not auto-fetch public pricing during a client report unless the source and
timestamp are preserved. Provider invoices, contract terms, and cloud cost
calculator exports are more stable evidence than live pricing pages.

## Token Cost Formula

Use the provider invoice as the final source of truth. For pilot estimates, use:

```text
request_cost =
  (input_tokens / 1_000_000 * input_price_per_1m_tokens)
  + (output_tokens / 1_000_000 * output_price_per_1m_tokens)
```

Monthly baseline cost:

```text
monthly_requests =
  users * requests_per_user_per_day * working_days_per_month

baseline_monthly_input_tokens =
  monthly_requests * baseline_average_input_tokens

baseline_monthly_output_tokens =
  monthly_requests * baseline_average_output_tokens

baseline_monthly_llm_cost =
  (baseline_monthly_input_tokens / 1_000_000 * baseline_input_price)
  + (baseline_monthly_output_tokens / 1_000_000 * baseline_output_price)
```

Pilot cost with local routing:

```text
pilot_cloud_requests =
  monthly_requests * (1 - local_route_share)

pilot_monthly_llm_cost =
  sum(cost_per_route_i * route_share_i)
```

Include the gateway cost:

```text
incremental_operations_cost =
  operations_hours_per_month * internal_hourly_cost

pilot_total_monthly_cost =
  pilot_monthly_llm_cost
  + azure_or_aws_monthly_cost
  + incremental_operations_cost
```

## Example Calculation

Replace these values with client measurements before sharing a commercial
claim.

| Assumption | Value |
| --- | ---: |
| Users | 25 |
| Requests per user per day | 20 |
| Working days per month | 20 |
| Monthly requests | 10,000 |
| Average input tokens | 2,000 |
| Average output tokens | 800 |
| Baseline blended cost per 1M input tokens | EUR 5.00 |
| Baseline blended cost per 1M output tokens | EUR 15.00 |
| Local route share | 45% |
| Cheaper cloud route share | 35% |
| Premium cloud route share | 20% |
| Azure/AWS gateway cost | EUR 90/month |
| Operations cost | EUR 100/month |

Baseline:

```text
input_cost = 10,000 * 2,000 / 1,000,000 * 5.00 = EUR 100
output_cost = 10,000 * 800 / 1,000,000 * 15.00 = EUR 120
baseline_monthly_llm_cost = EUR 220
```

Pilot example:

```text
local_route_cost = EUR 0 provider cost
cheaper_cloud_cost = EUR 38
premium_cloud_cost = EUR 44
pilot_provider_cost = EUR 82
pilot_total_monthly_cost = 82 + 90 + 100 = EUR 272
net_monthly_savings = 220 - 272 = -EUR 52
```

Interpretation: with this small volume, the pilot is not justified by direct
token savings alone if a dedicated cloud gateway and operations cost are fully
allocated. It can still be justified by privacy, auditability, sovereignty,
latency, or reliability. Direct savings improve when request volume is higher,
the local route share is higher, or the gateway is shared across more users.

Higher-volume example using the same routing mix:

```text
monthly_requests = 100,000
baseline_monthly_llm_cost = EUR 2,200
pilot_provider_cost = EUR 820
pilot_total_monthly_cost = 820 + 90 + 100 = EUR 1,010
net_monthly_savings = 2,200 - 1,010 = EUR 1,190
savings_rate = 1,190 / 2,200 * 100 = 54.1%
```

## Avoided 429 Calculation

Track `429` responses before and after adaptive routing.

```text
avoided_429 =
  baseline_429_count
  - pilot_429_count
```

If the adaptive router records dry-run or shadow decisions, count an avoided
`429` only when all three conditions are true:

- the baseline route was already near its RPM or TPM limit;
- the adaptive route selected another healthy route;
- the request succeeded without hitting a provider rate limit.

Use this table for the client report:

| Period | Requests | 429 count | 429 rate | Notes |
| --- | ---: | ---: | ---: | --- |
| Baseline | TBD | TBD | TBD | Before adaptive routing |
| Pilot | TBD | TBD | TBD | With adaptive routing |
| Difference | TBD | TBD | TBD | Avoided errors |

Formula:

```text
429_rate = 429_count / total_requests * 100
```

## Cloud Hosting Cost

For OVHcloud, Hetzner, AWS, Azure, or a local server, include the recurring
components that exist in the selected architecture. Keep provider names in the
input JSON as labels; the calculator does not need provider-specific code.

| Component | Azure example | AWS example | OVHcloud / Hetzner example | Monthly estimate |
| --- | --- | --- | --- | ---: |
| CPU VM or container | VM, Container Apps, AKS node | EC2, ECS, EKS node | VPS, dedicated server, cloud VM | TBD |
| Optional GPU | NC-family VM | g5/g6 instance | GPU server or GPU cloud instance | TBD |
| Disk or volume | Managed Disk | EBS | Block storage or local NVMe | TBD |
| Load balancer or reverse proxy | Application Gateway, Load Balancer | ALB, NLB | Nginx, HAProxy, managed LB | TBD |
| Secrets | Key Vault | Secrets Manager, KMS | Vault, sealed env file, provider secrets | TBD |
| Logs and monitoring | Monitor, Log Analytics | CloudWatch | Netdata, Grafana, provider logs | TBD |
| Security services | Defender for Cloud | GuardDuty, Security Hub | Firewall, IDS, backup monitoring | TBD |
| Backups | Backup vault | AWS Backup | Snapshots, backup storage | TBD |
| Network egress | Bandwidth | Data transfer | Included quota or metered egress | TBD |

For a one-day pilot, prefer a small CPU gateway unless local model inference
quality is part of the test. GPU cost can dominate the economics quickly.
